## üéØ Prinsip Utama

**Aturan Rotasi:**
> Jika Try-Out Sesi 1 menggunakan materi X di kategori tertentu, maka **Sesi 2, 3, 4 dst** tidak boleh menggunakan materi X lagi sampai minimal **4 minggu** (8 Try-Out).

**Contoh:**
- Try-Out Minggu 1 (Rabu): PU pakai "Sebab-Akibat"
- Try-Out Minggu 1 (Minggu): PU **TIDAK BOLEH** pakai "Sebab-Akibat"
- Try-Out Minggu 2-4: PU **TIDAK BOLEH** pakai "Sebab-Akibat"
- Try-Out Minggu 5: PU **BOLEH** pakai "Sebab-Akibat" lagi

**Tujuan:**
- Hindari kebosanan user
- Latih semua aspek materi
- Fair untuk semua peserta (tidak bisa "hafal pola")

### üî¢ **PK ‚Äì Pengetahuan Kuantitatif** (20 soal, 30 menit)

**Ciri & Syarat Soal PK:**
- ‚úÖ Tipe **hitungan cepat & logis**
- ‚úÖ **Hitungan tidak panjang** (bisa dikerjakan manual)
- ‚úÖ **Bisa dikerjakan manual** (tidak butuh kalkulator rumit)
- ‚úÖ Fokus pada **konsep**, bukan rumus berat
- ‚úÖ Waktu ideal per soal: **1.5 menit**
- ‚úÖ **WAJIB ADA TRIK CEPAT** ‚ö° (sangat penting!)

**‚ö° Prinsip Khusus SNBT:**
> Soal PK dirancang terlihat sulit, tapi **PASTI ada trik cepat**!  
> Angka dipilih secara **strategis** (bukan random) untuk memicu trik tertentu.

**Ciri Soal dengan Trik Cepat:**
- Angka "bulat" atau proporsional (2, 3, 5, 10, 25, 50, 100)
- Perbandingan sederhana (2:3, 3:5, 1:2:3)
- Pattern yang bisa dieliminasi
- Dekat dengan perfect square (96, 99, 101, 104)
- Bisa diselesaikan dengan **logika**, bukan hitung panjang

**Contoh Trik:**
- Eliminasi opsi (digit terakhir)
- Perkalian silang (butterfly method)
- Pattern recognition (a¬≤ - b¬≤)
- Estimasi cerdas (‚àö99 ‚âà 10)
- Selisih bagian (ratio)

See [SOAL-CREATION-GUIDE.md](SOAL-CREATION-GUIDE.md) untuk contoh lengkap!

#### ‚úÖ **WAJIB di Setiap Sesi** (minimal 10-12 soal):
1. **Hitung cepat** (aritmetika dasar) (3-4 soal)
2. **Perbandingan & rasio** (3-4 soal)
3. **Persamaan sederhana** (2-3 soal)
4. **Interpretasi angka** (pecahan, persen) (2-3 soal)

#### üîÑ **ROTASI** (8-10 soal, pilih 3-4 materi per sesi):
5. Aritmetika sosial
6. Fungsi
7. **Log, turunan, matriks** (level dasar) ‚Üê Opsional, bisa muncul atau tidak
8. **Barisan & deret**
9. Sistem persamaan linear
10. **Statistik dasar** (mean, median, modus)
11. **Geometri sederhana** (luas, volume)
12. **Peluang dasar**

**üìå Catatan Penting:**
- PK **TIDAK ADA** topik wajib spesifik (log/turunan/matriks optional)
- Yang wajib: **kemampuan hitung rasional & penalaran kuantitatif**
- Bisa tanpa log, bisa tanpa turunan, tapi **PASTI ada hitungan**

### **Helper Function - Generate Try-Out Questions:**

```typescript
// lib/tryout/material-rotation-helpers.ts

export interface MaterialUsage {
  tryout_date: string; // 'YYYY-MM-DD'
  test_type: 'tps' | 'literasi';
  category: string;
  subcategories_used: {
    wajib: string[];
    rotasi: string[];
  };
}

/**
 * Ambil history materi yang sudah dipakai 4 minggu terakhir (8 Try-Out)
 */
export async function getRecentMaterialUsage(
  category: string,
  test_type: 'tps' | 'literasi',
  lookbackWeeks: number = 4
): Promise<MaterialUsage[]> {
  const supabase = createClient();
  
  const cutoffDate = new Date();
  cutoffDate.setDate(cutoffDate.getDate() - (lookbackWeeks * 7));
  
  const { data, error } = await supabase
    .from('tryout_material_usage')
    .select('*')
    .eq('category', category)
    .eq('test_type', test_type)
    .gte('tryout_date', cutoffDate.toISOString().split('T')[0])
    .order('tryout_date', { ascending: false });
  
  if (error) throw error;
  return data || [];
}

/**
 * Pilih subcategories untuk Try-Out baru (hindari yang sudah dipakai)
 */
export async function selectRotatingSubcategories(
  category: string,
  test_type: 'tps' | 'literasi',
  allSubcategories: string[],
  requiredCount: number
): Promise<string[]> {
  // Ambil history 4 minggu terakhir
  const recentUsage = await getRecentMaterialUsage(category, test_type, 4);
  
  // Kumpulkan semua subcategories yang sudah dipakai
  const usedSubcategories = new Set<string>();
  recentUsage.forEach(usage => {
    usage.subcategories_used.rotasi.forEach(sub => usedSubcategories.add(sub));
  });
  
  // Filter subcategories yang belum dipakai
  const availableSubcategories = allSubcategories.filter(
    sub => !usedSubcategories.has(sub)
  );
  
  // Jika tidak cukup, reset dan pakai semua
  if (availableSubcategories.length < requiredCount) {
    console.warn(`Not enough unused subcategories for ${category}, resetting rotation`);
    return shuffleArray(allSubcategories).slice(0, requiredCount);
  }
  
  // Shuffle dan ambil sesuai kebutuhan
  return shuffleArray(availableSubcategories).slice(0, requiredCount);
}

/**
 * Generate Try-Out questions dengan sistem rotasi
 */
export async function generateTryoutQuestions(
  test_type: 'tps' | 'literasi',
  tryout_date: string
): Promise<{
  pu?: any[]; // 30 soal
  ppu?: any[]; // 20 soal
  pbm?: any[]; // 20 soal
  pk?: any[]; // 20 soal
  lbi?: any[]; // 30 soal
  lbe?: any[]; // 20 soal
  pm?: any[]; // 20 soal
}> {
  const supabase = createClient();
  const result: any = {};
  
  // Define categories based on test_type
  const categories = test_type === 'tps' 
    ? ['pu', 'ppu', 'pbm', 'pk']
    : ['lbi', 'lbe', 'pm'];
  
  for (const category of categories) {
    const config = CATEGORY_CONFIGS[category];
    
    // 1. Ambil soal WAJIB
    const { data: wajibQuestions } = await supabase
      .from('questions')
      .select('*')
      .eq('category', category)
      .in('subcategory', config.wajib)
      .limit(config.wajibCount);
    
    // 2. Pilih subcategories untuk rotasi (hindari yang sudah dipakai)
    const selectedRotatingSubcategories = await selectRotatingSubcategories(
      category,
      test_type,
      config.rotasi,
      config.rotatingSections
    );
    
    // 3. Ambil soal ROTASI dari subcategories terpilih
    const { data: rotasiQuestions } = await supabase
      .from('questions')
      .select('*')
      .eq('category', category)
      .in('subcategory', selectedRotatingSubcategories)
      .limit(config.rotasiCount);
    
    // 4. Gabungkan dan shuffle
    const allQuestions = [...(wajibQuestions || []), ...(rotasiQuestions || [])];
    result[category] = shuffleArray(allQuestions).slice(0, config.totalCount);
    
    // 5. Save material usage ke database
    await supabase.from('tryout_material_usage').upsert({
      tryout_date,
      test_type,
      category,
      subcategories_used: {
        wajib: config.wajib,
        rotasi: selectedRotatingSubcategories
      }
    });
  }
  
  return result;
}

// Helper: Shuffle array
function shuffleArray<T>(array: T[]): T[] {
  const shuffled = [...array];
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
  }
  return shuffled;
}
```

## üéØ Best Practices

### **1. Rotasi Minimum Gap:**
- **Ideal:** 4 minggu (8 Try-Out) sebelum materi muncul lagi
- **Minimum:** 2 minggu (4 Try-Out) jika stock soal terbatas

### **2. Balance Difficulty:**
- Setiap sesi harus ada mix: Easy (20%), Medium (40%), Hard (40%)
- Rotasi materi **TIDAK BOLEH** membuat satu sesi jadi "terlalu mudah" atau "terlalu susah"

### **3. Pre-Generation:**
- Generate Try-Out questions **1 minggu sebelumnya**
- Review manual untuk ensure quality & rotasi benar
- Save ke database dengan flag `is_published = false`

### **4. Monitoring:**
- Track user performance per subcategory
- Adjust rotasi jika ada subcategory yang terlalu sulit/mudah
- Quarterly review (setiap 3 bulan) untuk re-balance
